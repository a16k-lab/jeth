## 1. What was ADDED

### The dual-syntax mode system (the foundation)
- Mode is PER FILE: a file whose leading comments contain `// use @decorators` is legacy
  decorator mode; any other file is NATIVE mode. Native is a permissive superset during
  migration (strict decorator bans are deferred, see roadmap).

### Native declarations (items #1-#10, all byte-identical)
- `==` / `!=` on string/bytes (keccak-idiom desugar).
- `#name` members = Solidity `private` (mangle pre-pass `$p$C$x`, demangled in diagnostics);
  bare TS `private` keyword is banned loudly (JETH445, was silently ignored).
- bare `class` = the contract; `abstract class` = base; `type P = {...}` = struct;
  `interface I { m(): View<u256> }` = call-target interface; `enum` as before.
- Marker return types are the design language: `View<T>` / `Pure<T>` / `Payable<T>` on
  interface methods; `External<T>` / `Payable<T>` on contract methods (Payable implies external).
- `static` field = constant (with initializer) / immutable (without); `static` method / `static get`
  = class-level function; all read idiomatically as `ClassName.x` (site-aware rewrite).
- Mutability is INFERRED (no more @view/@pure/@read); `get` = enforced read-only at ANY
  visibility, and GET IS A MUST for read-only value-returning externals (JETH352 guides you).
- bare field = state; visibility axes are orthogonal: bare(internal) / `#`(private) /
  `External<T>` / `Payable<T>`.
- Native `receive()` / `fallback()` by name (`fallback(): Payable<...>` for payable);
  the silent-drop footgun was fixed on the way.

### Errors and events
- Member fields: `X: error<{ a: u256 }>` / `Y: event<{ who: indexed<address>; n: u256 }>`,
  raised `throw this.X({ a: 1n })` / `revert(this.X({...}))` / `emit(this.Y({...}))` (named args).
- FILE-LEVEL declarations (solc parity, errors 0.8.4+/events 0.8.22+):
  `type Insufficient = error<{ need: u256 }>;` at top level, raised BARE and POSITIONALLY:
  `revert(Insufficient(1n, a))` / `emit(Moved(msg.sender, a))`. Named args stay member-only
  (a bare-name named-arg form would ambiguate the existing struct-literal positional path).

### Libraries
- `static class` = library (native @library); `self` convention = native attached calls
  (first param named `self` makes `a.min(b)` attach, scoped to import edges).
- Deployable (delegatecall) libraries: a `static class` method returning `External<T>`;
  link symbols / artifacts all verified vs solc. `#` private library fns work.

### The multi-file import system (v1 -> v2 -> aliases -> v3)
- v1 bundling: `compile(entry, { sources })`, named relative imports only, export-gated,
  deps-first splicing with per-file diagnostic remapping; cycles/modes/dep-contracts guarded.
- v2 per-name scoping: a cross-file reference REQUIRES an import edge (JETH039); unexported
  declarations are private to their file; covers type positions, extends, @using, attachments.
- Convenience aliases: `import { A as B }` (position-preserving AST rename).
- **v3 per-file declaration scoping** (the big one, `16b5bd5`): dep top-level declarations
  alpha-rename to `$mN$Name`; every file's references resolve through its OWN scope. Two files
  may declare the SAME name (even unexported helpers); aliases disambiguate same-named exports.
  The ENTRY file is never renamed (single-file behavior invariant by construction). Hash
  boundaries always speak the SOURCE name (error/event selectors + topics, ABI names, library
  link symbols, all diagnostics). Loud guards for every rename hazard (import-vs-local,
  dual imports, builtin/marker/`Array` dep names, `$mN$` source identifiers, nested-decl shadows).
  Swept by a 6-lens 1047-case adversarial workflow: ZERO miscompiles / over-acceptances / crashes.
